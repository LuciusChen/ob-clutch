#+title: ob-clutch

Org-Babel integration for the [[https://github.com/LuciusChen/clutch][clutch]] database client.

* Installation

=ob-clutch= depends on the =clutch= package:

#+begin_src emacs-lisp
(use-package clutch)
(use-package ob-clutch
  :after clutch)
#+end_src

For native backends, installing =clutch= is enough.  Its package metadata pulls
in =mysql= and =pg= transitively.

Generic JDBC blocks require [[https://github.com/LuciusChen/clutch][clutch]]'s JDBC bridge as well:

#+begin_src emacs-lisp
(require 'clutch-db-jdbc)
#+end_src

* Supported block types

- =mysql=
- =postgresql=
- =sqlite=
- =clutch= (generic entry point)

Generic =clutch= blocks can target JDBC backends such as Oracle or SQL Server
with a =:backend= header argument.

* Saved connections and passwords

=:connection= resolves entries from =clutch-connection-alist=.  Saved
connections without an explicit =:backend= follow clutch's default and use
=mysql=.

Password resolution follows clutch's connection rules:

1. =:password=
2. =:pass-entry= via =auth-source-pass=
3. =auth-source-search= via =:host= / =:user= / =:port=

When a saved connection omits =:pass-entry=, =ob-clutch= defaults it to the
connection name so pass entries like =mysql/dev= can still match =dev=.

Saved connections and inline headers also honor Clutch transport keys:
=:ssh-host=, =:tramp=, and the longer =:tramp-default-directory= spelling.
When a block is executed from a TRAMP org buffer and no explicit transport is
configured, =ob-clutch= applies =clutch-tramp-context-policy= before connecting.

* Header arguments

- =:connection=
- =:backend=
- =:host=
- =:port=
- =:user=
- =:password=
- =:database=
- =:pass-entry=
- =:ssh-host=
- =:tramp=
- =:max-rows=

Backend-specific headers are passed through when relevant:

- PostgreSQL: =:schema=, =:tls=, =:sslmode=, =:query-timeout=
- MySQL: =:tls=, =:ssl-mode=
- JDBC: =:url=, =:sid=, =:catalog=, =:manual-commit=, =:rpc-timeout=

=:max-rows= truncates data rows after the query returns.  The default is
unlimited; customize =org-babel-clutch-max-rows= or set =:max-rows= per block.
Use =:max-rows nil= to force unlimited output for a single block.

* Examples

MySQL:

#+begin_src org
#+begin_src mysql :connection dev
SELECT 1;
#+end_src
#+end_src

PostgreSQL:

#+begin_src org
#+begin_src postgresql :host 127.0.0.1 :port 5432 :user postgres :database app :schema public :max-rows 20
SELECT now();
#+end_src
#+end_src

SQLite:

#+begin_src org
#+begin_src sqlite :database /tmp/demo.db
SELECT sqlite_version();
#+end_src
#+end_src

Generic clutch/JDBC:

#+begin_src org
#+begin_src clutch :backend oracle :connection prod-oracle
SELECT sysdate FROM dual
#+end_src
#+end_src

Inline JDBC:

#+begin_src org
#+begin_src clutch :backend oracle :host db.example.com :port 1521 :user scott :sid ORCL
SELECT sysdate FROM dual
#+end_src
#+end_src
